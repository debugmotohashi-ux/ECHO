ECHO — PWA構成版(案B)
========================

■ アップロード手順(1回だけ)
GitHubの ECHO リポジトリを開き、
「Add file」→「Upload files」で、以下の6ファイルを全部まとめてドラッグ/選択して
1回のコミットでアップロードしてください(index.htmlは上書き確認が出ます)。

  1. index.html            … アプリ本体(PWA対応版に更新)
  2. manifest.webmanifest  … アプリ情報(名前・アイコン・起動方法)
  3. sw.js                 … Service Worker(オフライン起動を実現)
  4. icon-192.png
  5. icon-512.png
  6. icon-maskable-512.png

■ Pixel(Android Chrome)でのインストール
1. デプロイ完了後、URLをChromeで開く(?v=7 などを付けてキャッシュ回避推奨)
2. 右上メニュー(⋮)→「アプリをインストール」(または「ホーム画面に追加」)
3. 以前作った古いショートカットは削除してOK
→ 以後、全画面のスタンドアロンアプリとして起動します

■ iPhoneでの変化
使い方はこれまで通り(Safari→共有→ホーム画面に追加)。
今回のService Worker追加により、iPhoneでも一度開いた後は
オフラインでアプリ自体を起動できるようになります。

■ オフライン動作の条件
- 初回だけはオンラインで開く必要があります(その時にアプリ・ライブラリ・
  モデルがすべて端末にキャッシュされます)
- 以後は機内モードでも: アプリ起動 → 録音 → 文字起こし まで完結
- Supabase同期だけはオンライン復帰時に自動実行されます

■ 今後の更新運用
- アプリの更新は今まで通り index.html の上書きだけでOK
- manifest / sw.js / アイコンは基本触りません
- もしsw.jsを更新した場合は、中の CACHE = 'echo-shell-v1' の数字を
  v2, v3...と上げるとキャッシュが確実に切り替わります
